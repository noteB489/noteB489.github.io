GitHub 저장소와 Codex를 연동하면 블로그 포스트 작성/수정을 훨씬 빠르게 진행할 수 있다.

## 1) 저장소 연결

1. `noteB489.github.io` 저장소를 Codex 작업 대상으로 선택한다.
2. `_posts` 폴더를 기준으로 글 생성 규칙(파일명, front matter 형식)을 확인한다.

## 2) 포스트 생성 요청

Codex에게 아래처럼 구체적으로 요청하면 된다.

- 오늘 날짜 기준으로 새 포스트 파일 생성
- 제목/카테고리 지정
- 본문 초안 작성

예: `_posts/2026-05-11-codex-github-posting.md` 파일을 만들고, 주제를 "github 연동해서 codex로 post 올리기"로 작성.

## 3) 확인 및 반영

1. 생성된 markdown의 제목, 날짜, 본문 구조를 검토한다.
2. 필요하면 문체/섹션을 수정한다.
3. 커밋 후 푸시하면 GitHub Pages에 반영된다.

## 4) 팁

- 요청 시 원하는 톤(간결/상세), 분량, 목차를 같이 주면 결과 품질이 높아진다.
- 이미지가 필요하면 `assets/images` 경로 기준으로 함께 요청하면 편하다.
- 마지막에 "SEO용 요약 2줄" 같은 후속 요청을 붙이면 실사용에 좋다.
